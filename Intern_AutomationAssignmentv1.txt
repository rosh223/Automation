Automation Assignment
General Instructions
- Framework: Playwright / Cypress (choose one)
- Follow the Page Object Model (POM) design pattern for UI automation
- Ensure assertions for:
  • UI element visibility and interaction
  • Functional flow and expected behaviors
  • API response validations and functional correctness

Application to be used for Automation
1.Access https://www.automationanywhere.com/products/enterprise/community-edition
2.Register using personal email ID to login to Automation Anywhere Community Edition version.

Use Case 1: Form with Rules Builder (UI Automation) 
Steps:
1. Log in to the Community Edition application.
2. Navigate to Automation and create a new Form.
3. Drag and drop at least two Textbox elements onto the canvas.
4. Set properties for each textbox (label, min/max length, hint text, tooltip, default value).
5. Save the form, then navigate to the Rules tab.
6. Create a new rule (e.g., Rule1) and verify it appears in expanded mode in the rules list.
7. Add a condition to the rule using one of the textbox elements with a condition type (e.g., Is Not Empty, Contains).
8. Add a second condition with AND mode.
9. Add an action to the rule (e.g., Set Value on the other textbox element).
10. Use the rule card context menu to add a second rule below (Rule2) and a third rule (Rule3).
11. Save the form and verify all rules persist. 
Expectations:
- Automate the full Rules Builder flow using POM.
- Assert the following:
  • Add Rule button is visible and functional
  • Rules are listed and displayed in expanded mode
  • Edit button is present on each rule card
  • Conditions are correctly configured (element, condition type, value input visibility)
  • AND/OR condition mode selection works
  • Actions (Set Value) are properly assigned to target elements
  • Context menu option Add Rule Below functions correctly
  • All rules (Rule1, Rule2, Rule3) are visible in the rules list after creation 
Use Case 2: Learning Instance API Flow (API Automation)
Steps:
1. Authenticate via API using provided credentials and capture the auth token.
2. Identify the Learning Instance API endpoint using the browser Network tab (navigate to the AI tab → Learning Instance in the application UI).
3. Create a Learning Instance with document type Invoice via API.
4. Validate the created instance.  Expectations:
- Identify all required API endpoints using the browser Network tab.
- Assert the following:
  • HTTP status codes (200 OK, 201 Created)
  • Response time (optional but preferred)
  • Response body schema and field-level checks (e.g., id, name, status)
  • Functional accuracy (instance created with correct data and status)
